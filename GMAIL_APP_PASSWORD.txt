═════════════════════════════════════════════════════════════════════════════════════
                            WHY APP PASSWORD?
═════════════════════════════════════════════════════════════════════════════════════

⚠️  DON'T use your regular Gmail password!

✅ App Password is better because:
   • Specific to this application
   • Can be revoked anytime
   • Limited permissions (email only)
   • Doesn't expose main Gmail password
   • Can create multiple for different apps
   • Google tracks usage for security
   • Easy to rotate for security

❌ Regular Gmail password:
   • Grants full account access
   • Can't be app-specific
   • Riskier if leaked
   • Not recommended for apps


═════════════════════════════════════════════════════════════════════════════════════
                        PREREQUISITE: 2-STEP VERIFICATION
═════════════════════════════════════════════════════════════════════════════════════

You MUST have 2-Step Verification enabled to use app passwords.

If not already enabled, do this first:

1. Go to: https://myaccount.google.com/security

2. Look for "How you sign in to Google"

3. Find "2-Step Verification"
   • If it says "OFF" → Click on it
   • If it says "ON" → Skip to next section

4. Click "Start setup" (if needed)

5. Verify your phone number:
   • Google sends SMS
   • Enter the code
   • Confirm

6. Add backup options:
   • Recovery email (important!)
   • Recovery phone
   • Backup codes (save these!)

7. Click "Enable"

✓ 2-Step Verification now active

Now you can generate app passwords!

                        TROUBLESHOOTING: APP PASSWORDS
═════════════════════════════════════════════════════════════════════════════════════

Problem: "Can't find App passwords option"
Solution:
  □ Go to: https://myaccount.google.com/security
  □ Look for "2-Step Verification" status
  □ If it says "OFF" → Enable it first
  □ Wait 1 minute
  □ Refresh page
  □ "App passwords" should now appear

Problem: "Getting error when trying to login with app password"
Solution:
  □ Double-check password has NO SPACES
  □ Copy exactly: a1b2c3d4e5f6g7h8
  □ Not: a 1 b 2 c 3 d 4 e 5 f 6 g 7 h 8
  □ Try generating a new one
  □ Make sure EMAIL_USER matches Gmail account

Problem: "I forgot the app password"
Solution:
  □ Go to https://myaccount.google.com/security
  □ Click "App passwords"
  □ You'll see list of all generated passwords
  □ You can't see the old one, but you can DELETE it
  □ Generate a new one
  □ Update .env file

Problem: "I see multiple app passwords"
Solution:
  □ You've generated passwords before
  □ Each one is for a different device/app
  □ Delete the old unused ones
  □ Use the latest one
  □ Or generate a new one specifically for Nodemailer

Problem: "Do I need a different password for production?"
Solution:
  □ Good practice: YES, use different credentials
  □ Go through steps again for production Google account
  □ Generate a new app password
  □ Store in production .env variables (Heroku, etc)
  □ This way: production and development are separate


═════════════════════════════════════════════════════════════════════════════════════
                        SECURITY BEST PRACTICES
═════════════════════════════════════════════════════════════════════════════════════

✅ DO:
  □ Use app passwords (not Gmail password)
  □ Generate separate passwords for each app
  □ Store password in .env (not in code)
  □ Add .env to .gitignore
  □ Use different passwords for production
  □ Enable 2-Step Verification
  □ Review "Your Security Events" monthly
  □ Delete unused app passwords
  □ Monitor Gmail login activity

❌ DON'T:
  □ Hardcode password in .js files
  □ Commit .env to Git
  □ Share password in messages/emails
  □ Use same password for multiple services
  □ Use your regular Gmail password
  □ Store password in comments
  □ Post password on GitHub


═════════════════════════════════════════════════════════════════════════════════════
                        MANAGING APP PASSWORDS
═════════════════════════════════════════════════════════════════════════════════════

View All Passwords:
  1. Go to: https://myaccount.google.com/apppasswords
  2. You'll see a table of all app passwords
  3. Shows: App name, Device type, Last used

Delete Password:
  1. Open app passwords list
  2. Find the one to delete
  3. Click the X button
  4. Confirm deletion
  (Useful if you think it's been compromised)

Regenerate:
  1. Delete the old one
  2. Generate new one
  3. Update in .env and redeploy
  4. Old password stops working immediately


═════════════════════════════════════════════════════════════════════════════════════
                        IF YOUR PASSWORD IS COMPROMISED
═════════════════════════════════════════════════════════════════════════════════════

If someone gets your app password:

1. Go to: https://myaccount.google.com/apppasswords
2. DELETE the compromised password immediately
3. It stops working right away
4. Generate a NEW one
5. Update .env with new password
6. Restart your server
7. Review "Security events" to see if misused
8. The attacker loses access
